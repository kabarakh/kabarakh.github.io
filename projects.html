LADXR Spoiler Log Interface

What is this?

So you might have heard of game randomizers. And you might have heard of Legend of Zelda Link's Awakening.

If both, you might want to check out LADXR, a randomizer for Link's Awakening.

When you create a ROM, you can also create a spoiler log. This can be in JSON - and this JSON can be read by LADXR Spoiler Interface!

Dice Rollers

What is this?

In the rules of tabletop games, dice rolls follow a specific notation. What I wanted to do was to take this notation (even in complicated cases), parse the input and use RNG to "roll" the dice. That way, I could keep my big dice bag at home while playing D&D - because I had the laptop on the table already because I didn't want to use paper based charsheets and stuff like that.

The first version was built in AngularJS, then an Angular (2+) version followed. Since then, building a dice roller is often my "hello world" for a new frontend technology. The most current ready one is built in Vue.

D&D Initiative Tool

What is this?

Another thing for my D&D playing and dungeon mastering. When duing combat, the participants are put in order by their initiative and act in order. And there are possibilities to change the order. I was too lazy to think about some way to keep track of everything on paper, and also the players need to see the initative.

The idea is to have this initiative tracker. And maybe have some kind of websocket connection for players with mobiles/laptops (TODO: Develop)

And for the vue edition I try to build a DM helper version which keeps track of the characters in groups (with dates we are playing) and some of their stats. And notes for the monsters. And maybe an integration with the dice roller. But I keep doing too complex stuff and fail building a working tool...

Other work (e.g. for punkt.de)

Codeception packages

At work, we tend do write automatic behaviour/frontend tests with codeception, and we like to use the Gherkin syntax to be forced to write easy understandable steps for complex tests.

When we started using this, we put the necessary steps within the test project, but that way we couldn't reuse them (other than copying, which would mean we lose newer developments in older projects). So I did what every open source dev does and created repositories for different codeception modules which extend them and implements gherkin steps.
